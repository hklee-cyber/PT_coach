## 1. 시스템 개요

NIMS PT 코칭 시스템은 뉴퍼센트 스파르타 학원의 **주간 PT 코칭 기록을 디지털화**하고, 축적된 데이터를 바탕으로 **AI 월간 보고서를 자동 생성**하는 웹 플랫폼입니다.

### 사용자 역할

| 역할 | 설명 |
|------|------|
| **관리자 (Admin)** | 시스템 전체 관리. 멘토·학생 관리, AI 보고서 생성·관리 |
| **멘토 (Mentor)** | 담당 학생의 주간 코칭 기록 입력 및 리포트 출력 |

### 전체 워크플로우

```
[로그인] → [역할 분기]
              ├─ 관리자 → 대시보드 → 멘토/학생/보고서 관리
              └─ 멘토   → 대시보드 → 학생 선택 → 코칭 입력 → 저장/출력
```

## 2. 로그인 및 회원가입

### 2-1. 로그인

1. 브라우저에서 시스템 주소에 접속합니다.(pt-coach-gray.vercel.app/login)
2. **이름** 입력란에 등록된 이름(성함)을 입력합니다.
3. **비밀번호**를 입력합니다.(멘토 초기비밀번호 000000 로그인 후 비밀번호 변경)
4. **[로그인하기]** 버튼을 클릭합니다.

> **주의:** 이름은 시스템에 등록된 정확한 성함을 입력해야 합니다.
> 계정이 없으면 관리자에게 문의하거나 아래의 회원가입 절차를 따릅니다.

### 2-2. 비밀번호 변경

멘토는 로그인 후 멘토 대시보드 내 **비밀번호 변경** 섹션에서 직접 변경할 수 있습니다.

## 3. 관리자 가이드

### 3-1. 관리자 대시보드

로그인 후 관리자는 **4개의 메뉴 카드**로 구성된 대시보드가 표시됩니다.

| 카드 | 기능 | 뱃지 표시 조건 |
|------|------|----------------|
| **관리자 관리** | 관리자 계정 추가/삭제 | — |
| **멘토 관리** | 멘토 계정 추가/삭제 |
| **학생 관리** | 전체 학생 목록 조회·관리 | — |
| **보고서 관리 센터** | 이번 달 월간 AI 보고서 생성·관리 | 미생성 보고서 수 |

> 카드 우상단에 **노란색 뱃지**가 표시되면 즉시 처리가 필요한 항목이 있다는 의미입니다.

---

### 3-2. 관리자 계정 관리

`관리자 관리` 카드 → `/admin/managers`

- 현재 등록된 관리자 목록을 확인합니다.
- 새 관리자를 추가하거나 기존 관리자를 삭제할 수 있습니다.

---

### 3-3. 멘토 관리

#### 멘토 계정 관리

- **멘토 목록**: 이름, 담당 학생 수, 가입일, 초기 비밀번호를 확인할 수 있습니다.
- **삭제**: 해당 멘토 행의 삭제 버튼으로 계정을 삭제합니다.

> **초기 비밀번호**는 000000. 멘토에게 안내해 주세요.

---

### 3-4. 학생 관리

`학생 관리` 카드 → `/admin/students`

#### 학생 목록 조회

- 전체 학생 목록(이름, 목표 대학, 담당 멘토, 상태)을 확인합니다.
- **재원 중(active)** / **퇴원(inactive)** 상태 구분이 가능합니다.

#### 학생 추가 및 삭제

- **[학생 추가]** 버튼으로 새 학생을 등록합니다.
  - 입력 항목: 이름, 목표 대학
- 학생 행의 **삭제** 버튼으로 학생을 삭제합니다.

#### 학생 상세 페이지 (`/admin/students/[id]`)

특정 학생을 클릭하면 상세 페이지로 이동합니다.

| 기능 | 설명 |
|------|------|
| **멘토 배정** | 담당 멘토 및 코칭 요일/교시 지정 |
| **상태 변경** | 재원 중 ↔ 퇴원 전환 |
| **코칭 히스토리 조회** | 해당 학생의 전체 코칭 기록 열람 |
| **Word 전체 내보내기** | 누적 코칭 기록을 Word 파일로 일괄 다운로드 |

---

### 3-5. 보고서 관리 센터

`보고서 관리 센터` 카드 → `/admin/reports`

**이번 달 월간 AI 보고서**를 생성하고 관리하는 핵심 기능입니다.

#### 대시보드 현황 카드

| 카드 | 설명 |
|------|------|
| **이번 달 보고서 현황** | 전체 진행률(%) 및 완료/전체 인원 |
| **보고서 완료** | 이번 달 보고서가 생성된 학생 수 |
| **상담 필요** | 이번 달 코칭 기록이 0건인 학생 수 |

#### 보고서 생성

**단일 생성:**
1. 학생 행의 오른쪽 **[생성]** 버튼을 클릭합니다.
2. AI가 해당 학생의 누적 코칭 기록을 분석하여 보고서 초안을 자동 생성합니다.
3. 완료 시 행이 자동으로 확장되며 미리보기가 표시됩니다.

**일괄 생성:**
1. 체크박스로 여러 학생을 선택합니다. (헤더 체크박스로 전체 선택 가능)
2. 상단 **[선택 학생 일괄 생성]** 버튼을 클릭합니다.
3. 선택된 학생 순서대로 순차 처리됩니다.

> 이번 달 코칭 기록이 없는 학생은 **[생성]** 버튼이 비활성화**됩니다. 먼저 코칭 기록 입력이 필요합니다.

#### 보고서 편집 및 저장

보고서 생성 후 **[보기]** 버튼을 클릭하면 확장 영역이 열립니다.

| 탭 | 기능 |
|----|------|
| **미리보기** | 마크다운 형식으로 렌더링된 보고서 확인 |
| **직접 편집** | 텍스트 에디터에서 내용 직접 수정 |

- **[저장]**: 수정 내용을 데이터베이스에 저장합니다.
- **[재생성]**: AI로 보고서를 다시 생성합니다. (기존 내용 덮어씀)
- **[PDF 출력]**: 새 창에서 인쇄 최적화 레이아웃으로 PDF 출력합니다.

#### 필터 기능

| 필터 | 표시 내용 |
|------|----------|
| **전체** | 모든 학생 |
| **완료** | 보고서가 생성된 학생만 |
| **미생성** | 보고서가 없는 학생만 |

## 4. 멘토 가이드

### 4-1. 멘토 대시보드

로그인 후 멘토는 아래 3개 섹션으로 구성된 대시보드를 확인합니다.

#### 오늘의 시간표

- 오늘 요일에 배정된 **담당 학생 슬롯(교시)**이 자동으로 표시됩니다.
- 학생 이름을 클릭하면 바로 코칭 입력 페이지로 이동합니다.

#### 비밀번호 변경

- 현재 비밀번호와 새 비밀번호(6자리)를 입력하여 변경합니다.

#### 전체 담당 학생

- 배정된 모든 학생의 이름, 목표 대학, 코칭 요일/교시를 확인합니다.
- 학생 이름 클릭 시 해당 학생의 코칭 입력 페이지로 이동합니다.

---

### 4-2. PT 코칭 입력

`/mentor/students/[id]`

학생의 주간 코칭 내용을 입력하는 핵심 페이지입니다.

#### 상단 정보 입력

| 항목 | 설명 |
|------|------|
| **날짜** | 코칭 일자 (기본값: 오늘) |
| **멘토** | 현재 로그인한 멘토명 (자동) |
| **입시유형** | 정시 / 수시 / 미정 선택 |
| **성적** | 현재 성적 입력 (예: 국2 수3 영2) |

#### 코칭 테이블 구조

테이블은 **3개 구분(학습 전략 / Review & Feedback / Action Plan)** 과 **5개 과목(국어 / 수학 / 영어 / 탐구1 / 탐구2)** 로 구성됩니다.

| 구분 | 항목 |
|------|------|
| **학습 전략** | 등급목표, 교재/인강/현강, 과목별 공부전략, 플래너 체크(순공시간) |
| **Review & Feedback** | 지난주 학습진도, 복습&습관, 자기점검(Test) |
| **Action Plan** | 다음 주 계획, 집중훈련 |

> **탐구 과목명 입력:** 탐구1·2 열 헤더의 입력란에 과목명(예: 생명과학, 지구과학)을 직접 입력합니다.

#### 각 셀 입력 방법

1. 셀을 클릭하면 텍스트 입력 영역이 활성화됩니다.
2. 내용을 자유롭게 입력합니다. (줄바꿈 가능)
3. 다른 셀로 이동하면 자동으로 내용이 유지됩니다.

---

### 4-3. 코칭 기록 관리

#### 이전 자료 불러오기

직전 회차의 코칭 기록을 현재 폼으로 자동 로드하는 기능입니다.

1. 툴바의 **[이전 자료 불러오기]** 버튼을 클릭합니다.
2. 가장 최근 기록의 모든 내용이 현재 폼에 덮어 씌워집니다.
3. 내용을 수정한 뒤 저장하면 새 회차로 기록됩니다.

> **주의:** 불러오기 실행 시 현재 입력 중인 내용이 **덮어써집니다.** 필요한 경우 먼저 저장 후 실행하세요.

#### 지난주 코칭 내용 보기

1. 툴바의 **[지난주 코칭 내용 보기]** 버튼을 클릭합니다.
2. 테이블 위에 직전 회차 기록이 **읽기 전용**으로 표시됩니다.
3. 현재 폼 작성 시 참고 자료로 활용하세요.
4. 다시 클릭하면 닫힙니다.

#### 과거 기록 목록 (히스토리 모달)

1. 툴바의 **[과거 기록 목록]** 버튼을 클릭합니다.
2. 모달에서 전체 코칭 기록 날짜 목록을 확인합니다.
3. 원하는 회차를 클릭하면 해당 기록이 현재 폼에 로드됩니다.
4. 각 회차 옆의 **Word 다운로드 버튼**으로 해당 기록을 Word 파일로 저장할 수 있습니다.
5. `Esc` 키 또는 배경 클릭으로 모달을 닫습니다.

#### 저장

- 상단 또는 하단의 **[저장]** 버튼을 클릭합니다.
- "✓ 저장되었습니다." 메시지가 표시되면 정상 저장된 것입니다.
- **같은 날짜로 저장 시**: 기존 기록이 덮어써집니다.
- **다른 날짜로 저장 시**: 새 회차 기록으로 추가됩니다.

---

### 4-4. 코칭 리포트 출력

#### 브라우저 출력 (인쇄/PDF 저장)

1. 툴바 또는 하단의 **[출력]** 버튼을 클릭합니다.
2. 브라우저 인쇄 창이 열립니다.
3. 대상 프린터를 **"PDF로 저장"** 으로 선택하면 PDF 파일로 저장됩니다.
4. 출력 시 불필요한 UI(버튼, 툴바 등)는 자동으로 숨겨지고 **코칭 테이블만 출력**됩니다.

> 출력물 상단에는 학생명, 담당 멘토, 코칭 일시, 목표 대학, 입시유형, 성적이 자동으로 표시됩니다.

#### Word 파일 내보내기

- 히스토리 모달에서 각 회차 옆의 **Word 아이콘 버튼**을 클릭합니다.
- 해당 회차 코칭 기록이 `.docx` 형식으로 다운로드됩니다.

## 6. 자주 묻는 질문 (FAQ)

**Q. 로그인이 안 됩니다.**
> A. 이름(성함)과 비밀번호를 정확히 입력했는지 확인하세요. 가입 후 관리자 승인이 되지 않은 경우에도 로그인이 불가합니다. 관리자에게 문의하세요.

**Q. 비밀번호를 잊어버렸습니다.**
> A. 관리자에게 연락하면 초기 비밀번호를 안내받을 수 있습니다. 이후 멘토 대시보드의 비밀번호 변경 기능을 사용하세요.

**Q. 코칭 기록을 잘못 입력해서 수정하고 싶습니다.**
> A. 히스토리 모달에서 해당 날짜 기록을 불러온 뒤 수정하고 **같은 날짜**로 저장하면 덮어써집니다.

**Q. 탐구 과목이 1개만 있는 학생은 어떻게 하나요?**
> A. 사용하지 않는 탐구 열(탐구1 또는 탐구2)은 비워두면 됩니다. 입력하지 않은 셀은 출력 시 공란으로 표시됩니다.

**Q. AI 보고서 생성 버튼이 비활성화되어 있습니다.**
> A. 해당 학생의 이번 달 코칭 기록이 없는 경우입니다. 멘토가 코칭 기록을 먼저 입력해야 합니다.

**Q. AI 보고서를 생성했는데 내용이 마음에 들지 않습니다.**
> A. 보고서 확장 영역의 **[직접 편집]** 탭에서 내용을 수정한 뒤 **[저장]**하세요. 또는 **[재생성]** 버튼으로 AI가 새로운 초안을 만들게 할 수도 있습니다.

**Q. 같은 학생에 대해 멘토가 여러 명 배정될 수 있나요?**
> A. 현재 시스템은 학생 1명당 1명의 멘토 배정을 기본으로 합니다. 변경이 필요한 경우 관리자가 학생 상세 페이지에서 멘토를 재배정할 수 있습니다.
